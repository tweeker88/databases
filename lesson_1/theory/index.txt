Функции СУБД:
1) Управление хранением
2) Управление буферами оперативной памяти
3) Управление транзакциями
4) Журнализация
5) Поддержка языков БД

12 правил Кодда
1) Вся хранимая информация представляется в виде двумерных таблиц
2) Доступ осуществляется через имя таблицы, первичного ключа и имени колонки
3) Пустые значения NULL означают "отсутствие информации". NULL не равен пустой строке, пробелу или 0
4) Информация с описанием БД хранится в самой БД в виде системных таблиц
5) СУБД должна поддерживать единый язык для определения данных, манипуляций с данными, транзакций и обеспечения целостности данных
6) СУБД должна поддерживать альтернативный способ отображения различных логических комбинаций данных:
    виртуальные таблицы - представления (view)
7) Все операции над данными определяеются как операции над множеством. Записи БД - это элементы двумерного множества,
    над которым определены операции реляционной алгебры.
8) Физическая организация данных отделена от приложений, оперирующих с БД.
9) Логическая организация данных отделена от приложений.
10) Ограничения целестности данных поддерживается на стороне СУБД.
11) Приложение не знает о сетевой организации БД
12) Изменения БД должно осуществляться через язык программирования.

Реляционная алгебра
- Основана на теории множеств
- Представляет собой набор операторов, использующих отношения в качестве аргументов, и возвращает отношения в кач-ве результата.
- Является замкнутой, т.к. в кач-ве аргументов в реляционные операторы можно подставить другие реляц.операторы.

Понятия реляционной алгебры
1) Тип данных
2) Домен - столбец таблицы
3) Кортеж - запись в таблице
4) Атрибут - имя столбца
5) Отношения - таблица
6) Арное отношение - кол-во столбцов
7) Мощность отношения - кол-во записей

Свойство отношений
- Все кортежи уникальные
- Все значение атрибутов в отношении семантически атомарные
- Атрибуты и кортежи в отношении не упорядочены
- Отношение имеет атрибут или набор атрибутов, по которому однозначно идентифицируется кортеж
- Отношение должно содержать хоть один атрибут
- Отношение может не содержать кортежей

NULL
- не имеет типа
- может записываться в поля любого типа
- любая операция с NULL дает в результате NULL

Целостность БД
1) Целостность БД - сответсвие данных в БД заложенной логике, структуре и заданным правилам
2) Constraints - ограничения наложенные на возможные состояния БД
3) Целостность сущности - необходимость наличие первичного ключа
4) Ссылочная целостность - для каждого внешнего ключа должен существовать первичный ключ

Нормализация - исключить избыточность данных, которое является причиной аномалий, при добавление, удаление или изменения данных
1 правило - все атрибуты являются атомарными
2 правило - первичный ключ простой, если состовной, то не должно существовать частичной зависимости
3 правило - нет атрибутов находящихся в транзитивной зависимости
